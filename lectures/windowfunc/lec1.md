# Лекция: Ранжирование данных через оконные функции в PostgreSQL

## 1. Введение в оконные функции

Оконные функции (window functions) в PostgreSQL — это мощный инструмент, который позволяет выполнять вычисления по набору строк, связанных с текущей строкой. В отличие от обычной агрегации с `GROUP BY`, оконные функции не сворачивают строки в одну, а возвращают значение для каждой строки, сохраняя детализацию.

**Синтаксис:**
```sql
function_name(expression) OVER (partition_clause order_clause frame_clause)
```

**Ключевые понятия:**
- **PARTITION BY** — делит строки на группы (окна). Если не указано, окно — вся таблица.
- **ORDER BY** — задает порядок строк внутри окна.
- **Frame** — определяет подмножество строк в окне (сдвиги, диапазоны).

---

## 2. Функции ранжирования

PostgreSQL предоставляет четыре основные функции для ранжирования:

| Функция | Описание | Поведение при одинаковых значениях |
|---------|----------|-----------------------------------|
| `ROW_NUMBER()` | Нумерует строки по порядку | Уникальный номер для каждой строки, даже при равенстве |
| `RANK()` | Ранг с пропусками | Одинаковое значение = одинаковый ранг, следующий ранг пропускается |
| `DENSE_RANK()` | Плотный ранг без пропусков | Одинаковое значение = одинаковый ранг, следующий ранг без пропусков |
| `NTILE(n)` | Делит окно на n групп (квантили) | Распределяет строки по корзинам максимально равномерно |

## 4. Функция ROW_NUMBER()

`ROW_NUMBER()` присваивает уникальный номер каждой строке в окне. Если есть одинаковые значения в `ORDER BY`, номера всё равно будут разными (порядок среди равных недетерминирован без дополнительных полей).

**Пример: Пронумеруем всех студентов по убыванию баллов:**
```sql
SELECT 
    student_name,
    subject,
    score,
    ROW_NUMBER() OVER (ORDER BY score DESC) as row_num
FROM exam_results;
```

**Результат:**
| student_name | subject    | score | row_num |
|--------------|------------|-------|---------|
| Анна         | Математика | 95    | 1       |
| Борис        | Физика     | 92    | 2       |
| Анна         | Физика     | 88    | 3       |
| Виктор       | Физика     | 88    | 4       |
| Борис        | Математика | 87    | 5       |
| Виктор       | Математика | 87    | 6       |
| Галина       | Физика     | 79    | 7       |
| Галина       | Математика | 76    | 8       |
| Дмитрий      | Математика | 65    | 9       |

**Важно:** У Анны и Виктора одинаковый балл (88), но номера разные.

**С группировкой по предметам:**
```sql
SELECT 
    student_name,
    subject,
    score,
    ROW_NUMBER() OVER (PARTITION BY subject ORDER BY score DESC) as subject_rank
FROM exam_results;
```

---

## 5. Функция RANK()

`RANK()` присваивает одинаковый ранг одинаковым значениям, но следующий ранг "пропускает" количество равных строк.

```sql
SELECT 
    student_name,
    subject,
    score,
    RANK() OVER (PARTITION BY subject ORDER BY score DESC) as rank
FROM exam_results;
```

**Результат (Математика):**
| student_name | score | rank |
|--------------|-------|------|
| Анна         | 95    | 1    |
| Борис        | 87    | 2    |
| Виктор       | 87    | 2    |
| Галина       | 76    | 4    |
| Дмитрий      | 65    | 5    |

Обратите внимание: после двух вторых мест следующее — четвёртое. Третье место пропущено.

---

## 6. Функция DENSE_RANK()

`DENSE_RANK()` ведёт себя так же, как `RANK()`, но без пропусков. Ранги идут подряд.

```sql
SELECT 
    student_name,
    subject,
    score,
    DENSE_RANK() OVER (PARTITION BY subject ORDER BY score DESC) as dense_rank
FROM exam_results;
```

**Результат (Математика):**
| student_name | score | dense_rank |
|--------------|-------|------------|
| Анна         | 95    | 1          |
| Борис        | 87    | 2          |
| Виктор       | 87    | 2          |
| Галина       | 76    | 3          |
| Дмитрий      | 65    | 4          |

Здесь после двух вторых мест идёт третье, без пропусков.

---

## 7. Сравнение ROW_NUMBER, RANK и DENSE_RANK

| Значение | ROW_NUMBER | RANK | DENSE_RANK |
|----------|------------|------|------------|
| 100      | 1          | 1    | 1          |
| 95       | 2          | 2    | 2          |
| 95       | 3          | 2    | 2          |
| 90       | 4          | 4    | 3          |

**Правило выбора:**
- Нужна уникальная нумерация → `ROW_NUMBER()`
- Нужны "олимпийские" места с пропусками → `RANK()`
- Нужны плотные места без пропусков → `DENSE_RANK()`

## 8. Функция NTILE()

`NTILE(n)` разбивает строки окна на n групп (квантилей) примерно равного размера. Часто используется для создания перцентилей, децилей, квартилей.

```sql
SELECT 
    student_name,
    subject,
    score,
    NTILE(4) OVER (ORDER BY score DESC) as quartile
FROM exam_results;
```

**Результат:**
| student_name | subject    | score | quartile |
|--------------|------------|-------|----------|
| Анна         | Математика | 95    | 1        |
| Борис        | Физика     | 92    | 1        |
| Анна         | Физика     | 88    | 1        |
| Виктор       | Физика     | 88    | 2        |
| Борис        | Математика | 87    | 2        |
| Виктор       | Математика | 87    | 3        |
| Галина       | Физика     | 79    | 3        |
| Галина       | Математика | 76    | 4        |
| Дмитрий      | Математика | 65    | 4        |

Строки распределены по 4 корзинам. При невозможности строгого равенства первые корзины получают на одну строку больше.

## 9. Практические примеры

### 9.1. Топ-2 студента по каждому предмету

```sql
WITH ranked AS (
    SELECT 
        student_name,
        subject,
        score,
        RANK() OVER (PARTITION BY subject ORDER BY score DESC) as rnk
    FROM exam_results
)
SELECT *
FROM ranked
WHERE rnk <= 2;
```

### 9.2. Разница между текущим баллом и лучшим по предмету

```sql
SELECT 
    student_name,
    subject,
    score,
    FIRST_VALUE(score) OVER (PARTITION BY subject ORDER BY score DESC) as max_score,
    FIRST_VALUE(score) OVER (PARTITION BY subject ORDER BY score DESC) - score as diff_from_max
FROM exam_results;
```

### 9.3. Нумерация с группировкой и сложным порядком

```sql
SELECT 
    student_name,
    subject,
    score,
    ROW_NUMBER() OVER (PARTITION BY subject ORDER BY score DESC, student_name) as rank_with_tiebreaker
FROM exam_results;
```

## 10. Производительность и индексы

Оконные функции требуют сортировки данных:
- `ORDER BY` в окне вызывает сортировку (или использует индекс)
- `PARTITION BY` группирует данные

**Советы:**
- Индекс на `(partition_col, order_col)` может значительно ускорить запрос
- Для больших таблиц избегайте `ORDER BY` без `PARTITION BY` на всей таблице
- Используйте `EXPLAIN ANALYZE` для анализа плана выполнения

```sql
-- Полезный индекс для запроса с PARTITION BY subject ORDER BY score DESC
CREATE INDEX idx_subject_score ON exam_results(subject, score DESC);
```

---

## 11. Ограничения и особенности

1. **Оконные функции нельзя использовать в `WHERE`** — только в `SELECT` и `ORDER BY`. Для фильтрации используйте CTE или подзапрос.
2. **Порядок вычисления:** оконные функции выполняются после `WHERE`, `GROUP BY`, `HAVING`, но до `DISTINCT`, `LIMIT`/`OFFSET`.
3. **Вложенность:** оконные функции нельзя вкладывать друг в друга напрямую, но можно через CTE.

```sql
-- Неправильно (не сработает):
SELECT ROW_NUMBER() OVER (ORDER BY SUM(score) OVER (PARTITION BY subject)) ...

-- Правильно:
WITH sums AS (
    SELECT subject, SUM(score) as total_score
    FROM exam_results
    GROUP BY subject
)
SELECT 
    subject,
    total_score,
    RANK() OVER (ORDER BY total_score DESC) as rank
FROM sums;
```

---

## 12. Заключение

Оконные функции для ранжирования — это элегантный и производительный способ решать задачи:
- **Рейтинги и лидерборды** — `RANK`, `DENSE_RANK`
- **Уникальные порядковые номера** — `ROW_NUMBER`
- **Квантили и перцентили** — `NTILE`
- **Сравнение с лучшими/худшими в группе** — в комбинации с `FIRST_VALUE`, `LAST_VALUE`

PostgreSQL реализует оконные функции с высокой производительностью, что делает их незаменимым инструментом для аналитических запросов и ETL-процессов.
